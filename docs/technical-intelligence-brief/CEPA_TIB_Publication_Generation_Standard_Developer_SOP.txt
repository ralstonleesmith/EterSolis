CEPA TECHNICAL INTELLIGENCE BRIEF PUBLICATION GENERATION STANDARD
Developer SOP for Automated CEPA Technical Intelligence Brief Production
Flagship Release-Aligned Developer Edition
SOP Version: 2.1
Alignment Baseline: CEPA Technical Intelligence Brief | Color & Chemicals Industry Edition | Issue 001 | 5 July 2026 | Flagship Release
Last Updated: 2026-06-30

Document purpose:
This SOP defines the standardized, automation-first method for generating CEPA Technical Intelligence Briefs and related CEPA publications at consistent expert quality. It is designed for developers building or maintaining the CEPA publication engine.

This version upgrades the prior developer SOP to match the final flagship release workflow. It adds controlled finalization, PDF outline/bookmark generation, clickable resource links, duplicate footer text-layer protection, QR crop validation, metadata hardening, external-source spot-check protocol, and QA report generation.

Governing principle:
CEPA publications must be generated from structured content, reusable templates, deterministic layout rules, automated audits, and manual rendered-page review. The final PDF must never be manually assembled page by page.

Controlled exception:
A controlled post-processing/finalization layer is permitted after deterministic PDF generation only when it performs release hardening actions that are themselves scripted, auditable, and repeatable. Approved finalization actions include:
- removing duplicated footer text-layer artifacts from legacy or imported PDFs;
- adding or rebuilding PDF bookmarks/outlines;
- adding clickable link annotations over approved QR/resource blocks;
- setting or correcting PDF metadata;
- linearizing or normalizing the PDF structure where supported;
- generating a final release QA report.

The finalization layer must never rewrite substantive page content, alter claims, move design elements manually, or conceal visual defects.

Target standard:
- A4 portrait, fixed-layout publication quality.
- 40 pages for flagship issues unless explicitly approved otherwise.
- CEPA master header and footer.
- One and only one generated footer identity line per page in the visible text layer.
- No clipped logos.
- No distorted logos.
- No visible version labels.
- Version retained only in PDF metadata.
- No overlapping text or visuals.
- No cutoff content.
- No duplicated footer text.
- No broken QR code.
- QR codes on cover and back cover decode from final rendered PDF.
- Clickable link regions over cover/back-cover resource blocks.
- Complete PDF bookmarks for every page.
- No unresolved source codes.
- Complete source register.
- No draft language.
- No obsolete naming.
- Full render audit before release.
- Final QA report produced with release evidence.

        |
        v

        |
        v

        |
        v

        |

Rules:
1. Do not manually place final PDF elements outside the template system.
2. Do not perform ad hoc PDF patching. Use only scripted finalization functions.
3. Every visual element must be controlled by reusable components and coordinates.
4. Every factual claim must map to a source code.
5. Every source code must resolve in the source register.
6. The build must fail if any release gate fails.
7. The finalizer may repair technical PDF structure, but it may not compensate for bad design generation.
8. A flagship release is not complete until the final QA report and release manifest exist.

4. PAGE GEOMETRY AND SAFE ZONES
================================================================================

Use A4 portrait:

PAGE_W_MM = 210
PAGE_H_MM = 297

Hard safe zones:

SAFE_LEFT_MM = 16
SAFE_RIGHT_MM = 194
SAFE_TOP_MM = 10
BODY_BOTTOM_MM = 268
FOOTER_RULE_MM = 274
FOOTER_TOP_MM = 276
FOOTER_BOTTOM_MM = 297

Rules:
1. No body content may enter the footer zone.
2. No table, chart, card, diagram, QR code, logo, or callout may enter y=276-297 mm.
3. Bottom callouts must sit above y=268 mm.
4. Header assets must have at least 2 mm clearance from page edges and header rule.
5. All logos must preserve aspect ratio.
6. No asset may be stretched, compressed, cropped, or clipped.
7. Link annotations may overlap QR/resource blocks but must not create visible rectangles.
8. Bookmarks/outlines must not affect page geometry.

6. LOGO HANDLING STANDARD
================================================================================

The CEPA horizontal logo must be padded before placement.

Required source assets:
- assets/logos/cepa_horizontal.png
- assets/logos/cepa_horizontal_padded.png
- assets/logos/cepa_stamp.png
- assets/logos/cepa_stamp_padded.png

Padding rule:
- Add 8-12% transparent padding on all sides.
- Lock aspect ratio.
- Never crop.
- Never stretch.
- Never place at page edge.
- Use padded assets only in headers and small stamp positions.

Header logo rule:
- Use cepa_horizontal_padded.png.
- Position: x=16 mm, y=12 mm.
- Max width: 70 mm.
- Max height: 16 mm.
- preserveAspectRatio=True.
- mask="auto".

Stamp logo rule:
- Use cepa_stamp_padded.png.
- Position: x=180 mm, y=12 mm.
- Max width: 16 mm.
- Max height: 16 mm.
- preserveAspectRatio=True.
- mask="auto".

Logo audit rule:
The rendered logo must be visually sharp at 200 DPI and 300 DPI. The horizontal logo and stamp must not be clipped in rendered PNGs, in page thumbnails, or in final PDF viewers.

7. TITLE PAGE STANDARD
================================================================================

The title page must use a premium, restrained, institutional layout.

Header:
  CEPA horizontal logo: x=16, y=12, w=70, h=16
  Color dots:           x=150, y=15, w=25, h=5
  Stamp:                x=180, y=12, w=16, h=16

Main title panel:
  x=16, y=64, w=168, h=128
  background: #F2F5F7
  no overlap with resource block

Title:
  x=24, y=86, w=132, h=42

Subtitle:
  x=24, y=132, w=132, h=18

Metadata table:
  x=24, y=158, w=132, h=28

Bottom-left strapline:
  x=20, y=220, w=100, h=26

Bottom-right resource block:
  x=144, y=202, w=42, h=58

Title-page QR / stamp block:
Use one vertical block. Do not float the stamp and QR separately.

Block outer box:
  x=144 mm, y=202 mm, w=42 mm, h=58 mm
  border: #D9E1E8, 0.45 pt
  background: white

Top accent line:
  x=148 mm, y=206 mm, w=34 mm, h=0.8 mm
  color: #FDBA12

Label:
  text: "CEPA RESOURCE ACCESS"
  x=148 mm, y=211 mm, w=34 mm, h=5 mm
  size: 6.8 pt
  uppercase
  centered

Stamp logo:
  x=155 mm, y=219 mm, w=20 mm, h=20 mm

QR code:
  x=153 mm, y=242 mm, w=24 mm, h=24 mm
  quiet zone included in QR image

Caption:
  text: "Scan for CEPA resources"
  x=148 mm, y=268 mm, w=34 mm, h=4 mm
  size: 5.8 pt
  centered

Rules:
1. Do not place the QR code in the main title panel.
2. Do not let the resource block cross the footer rule.
3. Do not use a heavy QR frame that competes with the cover.
4. QR must decode from the final PDF render.
5. The stamp and QR must align vertically to the same visual centerline.
6. A transparent clickable link annotation must be placed over the full resource block on the final PDF.

8. BACK COVER STANDARD
================================================================================

The back cover must close the publication with participation, correction, and resource-access controls.

Back cover required content:
- CEPA identity.
- Publication title or closing invitation.
- Participation routes.
- Correction policy.
- Prepared-by line.
- QR/resource block.
- Footer.

Recommended zones:
  CEPA mark / title area: x=16, y=20, w=178, h=52
  Participation block:   x=16, y=86, w=112, h=86
  QR resource block:     x=142, y=94, w=42, h=58
  Correction policy:     x=16, y=190, w=178, h=34
  Prepared-by line:      x=16, y=244, w=178, h=12
  Footer guard:          x=0, y=276, w=210, h=21

Back cover QR rule:
- The QR must decode from the final rendered PDF.
- The QR block must have a transparent clickable link annotation.
- The correction policy must not collide with the QR block or footer.
- The back cover must be included in the PDF outline/bookmarks.

9. PAGE 2 STANDARD: IDENTITY, SCOPE, EVIDENCE
================================================================================

Page 2 is the control page. It must never be crowded.

Required layout:

Title panel:
  "Publication Identity, Scope & Evidence Standard"

Subtitle:
  "Definition, boundary and claim confidence for public use"

Identity table:
  x=16, y=112, w=102, h=48

Issue explanation:
  x=126, y=112, w=68, h=48

Evidence cards:
  3 columns x 2 rows
  x positions: 16, 74, 132
  y positions: 172, 200
  card size: w=50, h=20

Control boxes:
  Claim control standard: x=16, y=226, w=178, h=12
  Not covered:            x=16, y=241, w=178, h=12
  Boundary note:          x=16, y=256, w=178, h=12

Evidence rule callout:
  Prefer x=16, y=252, w=178, h=18 only if the control boxes are shortened.
  Do not place any content at y=270+ unless verified clear of footer.

Page 2 rules:
1. Do not duplicate footer text.
2. Do not allow evidence cards to overlap control boxes.
3. Do not allow text to touch card borders.
4. Do not compress text below minimum readable size.
5. If page 2 is too dense, remove or shorten a control box rather than crowding.
6. Page 2 must pass the extracted-text footer count audit because it is vulnerable to duplicate footer artifacts in legacy inputs.


================================================================================
10. PAGE 29 STANDARD: DISCLOSURE COMPARABILITY LIMITS
================================================================================

Page 29 must be table-led and alignment-led.

Required layout:

Title panel:
  "Disclosure Comparability Limits"

Subtitle:
  "Public reporting is useful; benchmarking requires common boundaries"

Main disclosure table:
  x=16, y=112, w=178, h=72

Denominator table:
  x=16, y=198, w=178, h=48

Formula / principle box:
  x=20, y=252, w=170, h=14
  text centered
  no overlap with callout

Bottom callout:
  x=16, y=258, w=178, h=14
  only if formula box is omitted or enough space remains

Recommended flow:
1. Disclosure comparability table.
2. Denominator table.
3. One centered formula box:
   "Comparable benchmark = common denominator + common boundary + common product family + common reporting period + route evidence + confidence level"
4. Short callout only if there is at least 10 mm vertical separation from the formula box.

Rules:
1. Do not stack the formula box and callout if they cannot fit.
2. Do not allow table rows to drift out of alignment.
3. Do not allow the denominator table to collide with the formula box.
4. Use only one bottom emphasis element if vertical space is tight.

8. Keep title case consistent.

10. Avoid marketing tone.

all: clean validate build finalize render audit contact qa package

	rm -rf build/drafts build/finalized build/renders build/audit build/final
	mkdir -p build/drafts build/finalized build/renders build/audit/contact_sheets build/audit/qr_crops build/audit/reports build/final

validate:
	python src/validate_content.py --issue content/$(ISSUE)
	python src/audit_claims.py --issue content/$(ISSUE)

build:
	python src/build_publication.py --issue content/$(ISSUE) --out $(DRAFT_PDF)

finalize:
	python src/finalize_pdf.py --issue content/$(ISSUE) --infile $(DRAFT_PDF) --outfile $(FINAL_PDF)

render:
	python src/audit_visual.py --pdf $(FINAL_PDF) --out build/renders --dpi 200
	python src/audit_visual.py --pdf $(FINAL_PDF) --out build/audit/renders_300dpi --dpi 300

audit:
	python src/audit_pdf.py --pdf $(FINAL_PDF)
	python src/audit_sources.py --issue content/$(ISSUE) --pdf $(FINAL_PDF)
	python src/audit_bookmarks_links.py --issue content/$(ISSUE) --pdf $(FINAL_PDF)
	python src/audit_qr.py --pdf $(FINAL_PDF) --expected https://etersolis.com --out build/audit/qr_crops

contact:

qa:
	python src/generate_qa_report.py --issue content/$(ISSUE) --pdf $(FINAL_PDF) --out build/audit/reports/CEPA_TIB_Color_Chemicals_Flagship_QA_Report.txt

package:
	python src/package_release.py --pdf $(FINAL_PDF) --issue content/$(ISSUE) --qa build/audit/reports/CEPA_TIB_Color_Chemicals_Flagship_QA_Report.txt

from reportlab.lib.units import mm
from cepa_theme import *

def rect_mm(x, y, w, h):
    return x * mm, PAGE_H - (y + h) * mm, w * mm, h * mm

def wrap_title(title, max_chars=34):
    words = title.split()
    lines = []
    cur = ""
    for word in words:
        if len((cur + " " + word).strip()) <= max_chars:
            cur = (cur + " " + word).strip()
        else:
            if cur:
                lines.append(cur)
            cur = word
    if cur:
        lines.append(cur)
    return lines

def draw_header(c, issue):
    # Header logos must use padded images.
    c.drawImage(
        "assets/logos/cepa_horizontal_padded.png",
        16 * mm,
        PAGE_H - 28 * mm,
        width=70 * mm,
        height=16 * mm,
        preserveAspectRatio=True,
        mask="auto",
    )

    c.setFillColor(CEPA_GREEN)
    c.circle(150 * mm, PAGE_H - 18 * mm, 1.6 * mm, fill=1, stroke=0)
    c.setFillColor(CEPA_GOLD)
    c.circle(156 * mm, PAGE_H - 18 * mm, 1.6 * mm, fill=1, stroke=0)
    c.setFillColor(CEPA_BLUE)
    c.circle(162 * mm, PAGE_H - 18 * mm, 1.6 * mm, fill=1, stroke=0)

    c.drawImage(
        "assets/logos/cepa_stamp_padded.png",
        180 * mm,
        PAGE_H - 28 * mm,
        width=16 * mm,
        height=16 * mm,
        preserveAspectRatio=True,
        mask="auto",
    )

    c.setStrokeColor(BORDER)
    c.setLineWidth(0.35)
    c.line(16 * mm, PAGE_H - 32 * mm, 194 * mm, PAGE_H - 32 * mm)

def draw_footer(c, issue, page_num):
    # Footer must be generated exactly once per page.
    c.setStrokeColor(BORDER)
    c.setLineWidth(0.35)
    c.line(16 * mm, PAGE_H - 274 * mm, 194 * mm, PAGE_H - 274 * mm)

    c.setFont(FONT_BOLD, FOOTER_SIZE)
    c.setFillColor(SLATE)
    line1 = (
        f"{issue['series_name']} | {issue['edition_name']} | "
        f"{issue['main_handle']} | {issue['edition_handle']} | Page {page_num:02d}"
    )
    c.drawString(16 * mm, PAGE_H - 282 * mm, line1)

    c.setFont(FONT_REG, FOOTER_SIZE - 0.4)
    line2 = (
        f"Prepared by {issue['prepared_by']} | {issue['prepared_by_url']} | "
        f"{issue['document_code']}"
    )
    c.drawString(16 * mm, PAGE_H - 288 * mm, line2)

def draw_title_panel(c, title, label="CEPA TECHNICAL INTELLIGENCE BRIEF"):
    x, y, w, h = rect_mm(16, 40, 150, 34)
    c.setFillColor(PANEL)
    c.roundRect(x, y, w, h, 5 * mm, fill=1, stroke=0)

    c.setFillColor(SLATE)
    c.setFont(FONT_BOLD, 6.8)
    c.drawString(21 * mm, PAGE_H - 49 * mm, label)

    c.setFillColor(CEPA_NAVY)
    size = TITLE_LONG_SIZE if len(title) > 48 else TITLE_SIZE
    c.setFont(FONT_BOLD, size)

    lines = wrap_title(title, max_chars=34)
    ty = PAGE_H - 61 * mm
    for line in lines[:2]:
        c.drawString(21 * mm, ty, line)
        ty -= size * 1.05

File: src/finalize_pdf.py

Purpose:
Apply deterministic release hardening to the generated draft PDF.

Required finalizer actions:
1. Open draft PDF.
2. Confirm page count.
3. Confirm A4 portrait pages.
4. Set metadata:
   - title
   - author

Implementation skeleton:

import argparse
import fitz
import yaml
from pathlib import Path

A4_W_PT = 595.276
A4_H_PT = 841.890
TOL = 3

def load_yaml(path):
    with open(path, "r", encoding="utf-8") as f:
        return yaml.safe_load(f)

def mm_to_rect(x, y, w, h, page_height_pt=A4_H_PT):
    mm = 72 / 25.4
    return fitz.Rect(
        x * mm,
        y * mm,
        (x + w) * mm,
        (y + h) * mm,
    )

def build_toc(pages):
    toc = []
    for p in pages:
        page_num = p["page"]
        label = p.get("bookmark_title") or p.get("title") or f"Page {page_num:02d}"
        toc.append([1, f"{page_num:02d} {label}", page_num])
    return toc

def add_resource_links(doc, url):
    # Cover resource block.
    cover_rect = mm_to_rect(144, 202, 42, 58)
    doc[0].insert_link({"kind": fitz.LINK_URI, "from": cover_rect, "uri": url})

    # Back-cover resource block.
    last_rect = mm_to_rect(142, 94, 42, 58)
    doc[-1].insert_link({"kind": fitz.LINK_URI, "from": last_rect, "uri": url})

def finalize(issue_dir, infile, outfile):
    issue = load_yaml(Path(issue_dir) / "issue.yaml")
    pages = load_yaml(Path(issue_dir) / "pages.yaml")["pages"]

    doc = fitz.open(infile)

    if len(doc) != len(pages):
        raise RuntimeError(f"Page count mismatch: PDF has {len(doc)}, pages.yaml has {len(pages)}")

    for i, page in enumerate(doc, start=1):
        rect = page.rect
        if abs(rect.width - A4_W_PT) > TOL or abs(rect.height - A4_H_PT) > TOL:
            raise RuntimeError(f"Page {i} is not A4 portrait.")

    doc.set_toc(build_toc(pages))
    add_resource_links(doc, issue["public_url"])

    meta = doc.metadata or {}
    meta.update({
        "title": issue["series_name"],
        "author": issue["prepared_by"],
        "subject": issue["edition_name"],
        "creator": "CEPA automated publication generator + controlled finalizer",
        "keywords": f"{issue['document_code']}; {issue.get('metadata_version', '')}; CEPA; EterSolis; {issue.get('release_tier', '')}",
    })
    doc.set_metadata(meta)

    doc.save(outfile, garbage=4, deflate=True, clean=True)

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--issue", required=True)
    parser.add_argument("--infile", required=True)
    parser.add_argument("--outfile", required=True)
    args = parser.parse_args()
    finalize(args.issue, args.infile, args.outfile)

26. BOOKMARK AND LINK AUDIT
================================================================================

File: src/audit_bookmarks_links.py

Required audit:
- PDF has at least one outline/bookmark entry per page.
- Flagship 40-page releases must have exactly 40 top-level page bookmarks unless extra section bookmarks are intentionally configured.
- Bookmark page targets must resolve correctly.
- Page 1 must contain a URI link annotation to public_url.
- Page 40 must contain a URI link annotation to public_url.
- Link annotations must not be visible rectangles.
- Link annotations must not point to a draft or local file.

Implementation skeleton:

import argparse
import sys
import fitz
import yaml
from pathlib import Path

def fail(msg):
    print(f"FAIL: {msg}")
    sys.exit(1)

def load_yaml(path):
    with open(path, "r", encoding="utf-8") as f:
        return yaml.safe_load(f)

def audit(issue_dir, pdf_path):
    issue = load_yaml(Path(issue_dir) / "issue.yaml")
    pages = load_yaml(Path(issue_dir) / "pages.yaml")["pages"]
    expected_url = issue["public_url"]

    doc = fitz.open(pdf_path)
    toc = doc.get_toc()

    if len(toc) < len(pages):
        fail(f"Expected at least {len(pages)} bookmarks, found {len(toc)}")

    top_level = [entry for entry in toc if entry[0] == 1]
    if issue.get("release_tier") == "Flagship" and len(top_level) < 40:
        fail("Flagship release must include page-level bookmarks for all 40 pages.")

    for page_index in [0, len(doc) - 1]:
        links = doc[page_index].get_links()
        uris = [l.get("uri") for l in links if l.get("kind") == fitz.LINK_URI]
        if expected_url not in uris:
            fail(f"Page {page_index + 1} missing expected URI link: {expected_url}")

    print("PASS: bookmark and link annotation audit complete.")

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--issue", required=True)
    parser.add_argument("--pdf", required=True)
    args = parser.parse_args()
    audit(args.issue, args.pdf)

- pass
- pass_with_caveat
- unable_to_verify
- fail

A flagship issue cannot release with a "fail" spot-check for a material claim.

- flattening a PDF solely to evade text audit failures.

  --issue content/color_chemicals_issue_001 \
  --infile build/drafts/legacy_input.pdf \
  --outfile build/final/CEPA_Technical_Intelligence_Brief_Color_Chemicals_Issue_001_2026-07-05_Flagship.pdf
